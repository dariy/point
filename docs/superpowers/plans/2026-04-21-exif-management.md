# EXIF Management Implementation Plan

> **Status: COMPLETE** — All tasks implemented and pushed on 2026-04-21. Beads issue `point-8ob` closed.

**Goal:** Store an immutable original-EXIF backup on upload, write user-edited EXIF back to the image file on disk, and allow one-click revert to the original — all in pure Go with no system dependencies.

**Architecture:** Add `original_metadata TEXT` (populated at upload, never changed). All EXIF string values are sanitized to `[a-zA-Z0-9 ]` on both read and write. A new `exif_writer.go` uses `dsoprea/go-jpeg-image-structure/v2` + `dsoprea/go-exif/v3` to write IFD0 ASCII-string tags back into JPEG files atomically. Two new endpoints — `PUT /api/media/:id/exif` and `POST /api/media/:id/revert-exif` — validate input, write to file, and update the DB. The frontend EXIF panel's "Save EXIF" button switches to the new endpoint; a new "Revert to original" button is added.

**Tech Stack:** Go 1.25 + Echo v4, sqlc, `dsoprea/go-jpeg-image-structure/v2`, `dsoprea/go-exif/v3`, Vanilla JS SPA.

**Beads issue:** `point-8ob`
