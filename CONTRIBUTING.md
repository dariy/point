# Contributing to Point

First off, thank you for considering contributing to Point!

## Reporting Bugs

* Make sure you test against the latest version.
* Use the GitHub issue tracker to report bugs.
* Provide a clear and descriptive title.
* Include detailed steps to reproduce the issue.
* Describe the exact behavior you observed and the behavior you expected.

## Proposing Features

* Use the GitHub issue tracker to suggest new features.
* Provide a clear and detailed explanation of the feature and why it's needed.
* Discuss the feature in the issue before starting implementation.

## Pull Request Process

1. Provide a comprehensive summary of the changes in your pull request.
2. Ensure any new or changed functionality is accompanied by corresponding tests.
3. Make sure all existing tests pass. Code quality gates and test coverage thresholds are enforced. Follow the TDD approach and don't reduce existing test coverage.
4. Follow the existing coding style and conventions. Let the linters run cleanly.
5. The pull request will be merged once it has been reviewed and approved by maintainers.
